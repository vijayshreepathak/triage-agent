Feature Guide

Clinical Triage

Each message runs through an 11-node LangGraph pipeline: parse, extract signals, normalize symptoms, red-flag detection, urgency classification, conditional search, evidence merge, clinical explanation, confidence scoring, and structured response.

Dataset Cases

One hundred benchmark cases load in the sidebar. Search, select, or navigate with the previous and next controls. Free-text input is supported for ad hoc evaluation.

Safety Model

LLM proposes, rules dispose. Deterministic red-flag rules can only escalate urgency. Disclaimers are fixed in code. Confidence is computed heuristically, not self-reported by the model.

Evidence Retrieval

High-acuity presentations skip external search to reduce latency. Other cases may query configured search providers. Source URLs are taken verbatim from retrieval results.

History and Audit

Each triage run is stored with patient ID, urgency, confidence, and decision summary. Use the History tab to review prior runs and reload a case into the composer.

Access Control

When Clerk authentication is enabled, sign in to run triage, view history, and access dashboard metrics. Session tokens are verified server-side via JWKS.
